# 서블릿 -> JSP -> MVC 패턴
1. 서블릿은 JSP와 같은 뷰 템플릿 기능이 약해서 불편했다. (서블릿은 비즈니스 로직에 html 사용, JSP는 뷰 로직에 자바 사용)
2. JSP에 기능을 다 담자니, 코드가 하나의 JSP에 다 들어가니까 유지보수가 힘들었다.
3. 로직이 있는 쪽은 서블릿으로 해결하고, 뷰 템플릿은 JSP로 해결하는 방식이 MVC 패턴!
